# Research: How is user intent and design purpose represented in HCI research, interaction design, and UX methodology? Research: (1) Hierarchical Task Analysis (HTA) and how it decomposes user goals into sub-goals and operations. (2) User journey maps and experience maps — how they capture user intent, emotional states, and success at each touchpoint. (3) Scenario-based design (Carroll, Rosson) — how scenarios encode user goals, context, and success criteria. (4) Activity Theory applied to HCI — how it represents subject, object, tool, rules, community, and division of labor for understanding purposeful action. (5) Use cases and user stories — how they structure intent (As a [user], I want [goal], so that [benefit]). (6) Mental models — how the user's expected behavior of a system relates to design intent. (7) Contextual design (Beyer & Holtzblatt) — how contextual inquiry structures the understanding of user work context and intent. Output: how these HCI frameworks define 'what a design is trying to achieve' and what structural elements they use to represent it.

## Findings

### Output

In HCI research and UX methodology, user intent and design purpose are represented through various frameworks that operationalize 'what a design is trying to achieve' at different levels of granularity:

1. **Hierarchical Task Analysis (HTA)**: Defines design goals as the user’s overall objective (top-level task) decomposed into concrete, observable sub-goals and operations. It uses task trees and plan diagrams to link abstract intent to implementable actions, aiming to reduce breakdowns in the sequence of operations.

2. **User Journey and Experience Maps**: Frame design purpose as supporting progress through a multi-touch experience. They capture intent at each stage, emotional states, and success outcomes across touchpoints. Structural elements include personas, stages, touchpoints, emotional curves, and pain points.

3. **Scenario-Based Design (Rosson & Carroll)**: Represents intent through concrete narratives (scenarios) involving actors with motivations and context. The design goal is to realize these envisioned episodes successfully. Elements include settings, actors, motivations, sequences of events, and success criteria.

4. **Activity Theory**: Situates design purpose within purposeful human activity. A design aims to enable a subject (actor) to pursue a motive-oriented object while navigating mediating artifacts, rules, community, and division of labor. It highlights structural contradictions that the design should resolve.

5. **Use Cases and User Stories**: Represent intent as an actor’s goal the system must enable. User stories use a lightweight format (As a [user], I want [goal], so that [benefit]) to focus on value, while use cases provide detailed interaction flows, preconditions, and postconditions to ensure system requirements meet user goals.

6. **Mental Models**: Define design intent as the alignment of system behavior and architecture with the user’s internal expectations. Structural elements include conceptual maps of entities and relationships, and gap analyses between the user's model and the system model.

7. **Contextual Design (Beyer & Holtzblatt)**: Defines purpose as supporting real-world work and life intents discovered through field research. It uses models like the Day-in-the-Life, Sequence models (highlighting specific 'intents' and 'breakdowns'), and Flow models to represent user practice and design opportunities.

**Cross-Framework Synthesis**: 
While HTA and use cases focus on micro-level task flows, scenarios and Contextual Design provide meso-level contextual narratives. Journey maps offer macro-level temporal and emotional layering, and Activity Theory provides a systems-level view of socio-cultural motives. Common representational elements across these frameworks include actors (who), goals (what), tasks (how), context (where), and success criteria (how it worked).
